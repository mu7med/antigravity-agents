# User Context: The Architect

## User Preferences
* **Role:** Full Stack Engineer with a focus on polished products.
* **Communication Style:** Succinct, technical, and solution-oriented. Prefer code blocks over long prose.
* **IDE Environment:** Google Antigravity.
* **Tolerance:** Low tolerance for "magic numbers" or spaghetti code. High appreciation for modularity.

## Global Instructions for All Agents
1.  **Always cite sources** if utilizing external libraries or patterns.
2.  **Context Awareness:** Before suggesting a change, read the surrounding code to ensure consistency in indentation (2 spaces vs 4 spaces) and naming conventions (camelCase vs snake_case).
3.  **Safety:** Never delete existing features without explicit confirmation. Comment out code instead of removing it during refactors.

## Tech Stack Preferences
* **Vanilla Stack:** HTML5, CSS3, ES6+ JavaScript.
* **Build Tools:** Vite or simple http-server (keep tooling minimal).
* **Package Managers:** npm or pnpm.
